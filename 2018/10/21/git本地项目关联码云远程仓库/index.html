如果是第一次，在 git 环境配置完成后，我们要先配置一些基本信息，也就是码云账号的相关信息

基本信息配置完后，要在码云先新建远程仓库，这样本地才有的关联呀创建完成就可以开始关联本地项目了（搓手手

方法一

第一步：用 cd 命令进入到本地项目，初始化一下，把本地的项目变成 git 的本地仓库

第二步：将本地仓库与码云上刚新建的远程仓库关联起来

第三步：在准备 push 本地项目或进行其他操作之前，要拉一遍项目，确保和远程仓库的代码没有冲突，主要是为了覆盖本地的.git 文件夹，因为本地和线上是分别 init 的 算两个库，.git 文件夹不同算作两个不同的库

方法二

第一步：克隆线上空文件夹

第二步：把本地项目复制扔进刚克隆下来的线上仓库，注意不要覆盖掉.git 文件夹（.git 是隐藏文件 要是没 √ 可见隐藏文件看不见）

这里举例的情况是，本地有代码，线上仓库为空，第三步完成后直接推就好了。如果是线上有代码，换设备需要在本机新建项目，就直接 git clone 就好了

到这，就基本完成了本地与远程的关联，接下来要推代码合作开发什么的就是后面的事啦~~over
